# 鬼杀队 AI 团队展示网站 v3 — 产品拆解方案

**出品：** 蟲柱·胡蝶忍（产品经理）  
**日期：** 2026-03-16  
**目标受众：** 懂鬼灭 + 不懂 AI 的普通观众  
**结论预览：** 7个缺失维度 → 加入4个、跳过1个（转为融合）、合并2组 → **新增4个章节**

---

## 现有章节结构（v2 baseline）

| # | 章节 ID | 标题 | 定位 |
|---|---------|------|------|
| 1 | hero | 首屏 | 吸引注意 |
| 2 | team | 团队全貌 | 介绍成员 |
| 3 | workflow | 任务流转 | 展示协作过程 |
| 4 | brain | AI 大脑 | 技术能力 |
| 5 | mobile | 掌上指挥 | 使用场景 |
| 6 | why | 为什么 9 > 1 | 差异化 |
| 7 | case | 实战演练 | 案例演示 |

## Part 1：7 个缺失维度逐一判断

### 维度 1：能做什么（按场景能力清单）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐⭐ 极高 — 这是最核心的转化信息，用户最想知道"这能帮我做什么" |
| 单 HTML 实现难度 | 🟢 低 — 卡片/标签切换即可，无需复杂逻辑 |
| 建议 | **✅ 独立章节加入** |

**理由：** v2 完全没有告诉用户"鬼杀队能帮我做什么具体事情"。用户看完整个网站，印象可能只有"很酷"，但不知道它能帮自己写邮件、做PPT还是查资料。这是最严重的信息缺口。

---

### 维度 2：为谁设计（目标用户定义）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐ 高 — 帮观众判断"这是说我吗？" |
| 单 HTML 实现难度 | 🟢 低 — 人物画像卡片或痛点列表 |
| 建议 | **🔗 与维度3合并** |

**理由：** "为谁设计"单独成章会显得太单薄。但它与"解决什么问题"天然配对——先描述目标用户，再指出这类用户的痛点，形成"你是这种人吗 → 你有这些困扰吗"的叙事节奏，转化力更强。

---

### 维度 3：解决什么问题（痛点描述）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐⭐ 极高 — 痛点共鸣是激发兴趣最快的方式 |
| 单 HTML 实现难度 | 🟢 低 — 对比列表或引用框 |
| 建议 | **🔗 与维度2合并** |

**理由：** 同上。两个维度合并为一章"你是这种人吗？"，先共鸣（痛点），再点名（目标用户），最后引导继续看下去。

---

### 维度 4：记忆系统原理（四层架构可视化）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐ 低 — "四层架构"是技术词汇，非AI用户难以消化 |
| 单 HTML 实现难度 | 🟡 中 — 需要额外的可视化组件 |
| 建议 | **⏭️ 不独立成章，融入现有「AI大脑」章节** |

**理由：** 四层架构对普通用户来说是信息过载。但"鬼杀队拥有记忆系统，记得主公说过的每一件事"这个**结果**是有价值的。建议在现有 brain 章节里用1-2句话说明"天音有记忆，不会忘事"，不做技术展开。如果未来有技术向版本再补充。

---

### 维度 5：质量门控原理（为什么每份成果要经过审核）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐⭐ 极高 — 直接建立信任，解决"AI会不会乱说"的最大顾虑 |
| 单 HTML 实现难度 | 🟢 低 — 简洁流程图或步骤展示 |
| 建议 | **🔗 与维度6合并** |

**理由：** 质量门控是机制，信任感是结果。两者天然是"因为有了XX机制，所以你可以信任我们"的因果关系，合并后论证逻辑更完整。

---

### 维度 6：信任感/可靠性（真实案例、纠错机制）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐⭐ 极高 — 没有信任就没有行动，这是转化的核心杠杆 |
| 单 HTML 实现难度 | 🟡 中 — 需要真实案例内容，但展示形式不复杂 |
| 建议 | **🔗 与维度5合并** |

**理由：** 同上。"义勇审核机制（质量门控）+ 真实纠错案例（信任感）"合并为"我们为什么靠谱"，一章解决"用户最大顾虑"。

---

### 维度 7：怎么开始（入口/联系方式）

| 评估项 | 结论 |
|--------|------|
| 对小白观众的价值 | ⭐⭐⭐⭐⭐ 极高 — 没有 CTA 的展示网站等于白做 |
| 单 HTML 实现难度 | 🟢 极低 — 一个 CTA 区块 |
| 建议 | **✅ 独立章节加入，作为终章** |

**理由：** 目前 v2 完全没有引导用户"下一步做什么"。观众看完很可能感叹一句"好厉害"然后关掉页面。必须加一个明确的落地点。

---

### Part 1 决策摘要

| 维度 | 决策 | 去向 |
|------|------|------|
| 能做什么 | ✅ 加入 | 独立章节 B |
| 为谁设计 | 🔗 合并 | 与维度3合并 → 章节 A |
| 解决什么问题 | 🔗 合并 | 与维度2合并 → 章节 A |
| 记忆系统原理 | ⏭️ 跳过 | 融入现有 brain 章节一句话 |
| 质量门控原理 | 🔗 合并 | 与维度6合并 → 章节 C |
| 信任感/可靠性 | 🔗 合并 | 与维度5合并 → 章节 C |
| 怎么开始 | ✅ 加入 | 独立章节 D（终章） |

**净增章节数：4 个新章节**

## Part 2：整合方案 — 新章节设计

### 全站章节新顺序

```
[1] hero（保留，建议小修）
[2] ★ NEW: 你有这些困扰吗？（为谁设计 + 解决问题）
[3] ★ NEW: 鬼杀队能做什么？（能力清单）
[4] team — 团队全貌（保留）
[5] workflow — 任务流转（保留）
[6] brain — AI 大脑（保留，加记忆系统一句话融入）
[7] mobile — 掌上指挥（保留）
[8] why — 为什么 9 > 1（保留）
[9] case — 实战演练（保留）
[10] ★ NEW: 为什么可以信任我们？（质量门控 + 信任感）
[11] ★ NEW: 怎么开始（CTA）
```

---

### 章节 A：你有这些困扰吗？

**ID:** `pain`  
**位置：** hero 之后，能力清单之前（第2屏）  
**对应维度：** 为谁设计 + 解决什么问题  

**核心信息（用户看完应该得到的认知）：**
> "这个东西是为像我这样的人做的，而且它能解决我真实遇到的问题。"

**内容结构：**
1. **上半部分 — 痛点共鸣区**（大字，直击内心）
   - 「每次写方案都要改八遍？」
   - 「查资料找半天，不知道该信哪个？」
   - 「一件事要跑好几个 AI 工具才能搞定？」
   - 「重要文件发出去才发现有错？」
2. **过渡语句：** "如果你说了三个'对'，鬼杀队就是为你准备的。"
3. **下半部分 — 目标用户定义**（3类人画像卡片）
   - 独立工作者：一个人撑起多条业务线
   - 小团队负责人：人少事多，每个决策都不敢出错
   - 内容/知识工作者：每天要生产高质量文字/分析内容

**叙事节奏：** 痛点触发情绪 → 确认身份 → 自然滑入下一章（能做什么）

---

### 章节 B：鬼杀队能做什么？

**ID:** `capabilities`  
**位置：** 章节 A 之后（第3屏）  
**对应维度：** 能做什么

**核心信息（用户看完应该得到的认知）：**
> "原来它能帮我做这么具体的事，而且按照我的场景分门别类，找起来很方便。"

**内容结构：**
- **场景标签切换**（Tab 或滑动切换）：
  - 📝 写作与内容（邮件、方案、报告、文案）
  - 🔍 调研与分析（竞品、行业、数据解读）
  - 💻 技术与代码（脚本、部署、自动化）
  - 📊 决策与战略（拆解问题、框架建议）
  - 🗓️ 日常管理（日程、提醒、信息整理）
- 每个场景下列 4-6 个具体任务，用动词开头（"帮你写 → 帮你找 → 帮你做"）
- **底部固定小字：** "以上每个任务，都由专属柱处理，经义勇审核后交付。"

**设计要点：**
- 标签切换必须平滑，不能全部展开（信息过载）
- 每个任务描述不超过12字，语言直白不用技术词汇
- 使用鬼灭主题图标，但旁边配上普通人能懂的emoji

---

### 章节 C：为什么可以信任我们？

**ID:** `trust`  
**位置：** 实战演练之后（第10屏）  
**对应维度：** 质量门控原理 + 信任感/可靠性

**核心信息（用户看完应该得到的认知）：**
> "这不是一个会乱说话的 AI，它有自己的内审机制，输出之前有人把关。"

**内容结构：**
1. **上半部分 — 质量门控流程**（3步可视化）
   ```
   柱执行 → 义勇审核 → 忍用户体验审查 → 天音交付
   ```
   - 每步配 1 行说明
   - 强调：任何输出发出前必须经过两道审核
2. **中部 — 义勇的审核标准**（折叠展开，好奇心驱动）
   - 逻辑是否自洽？
   - 事实是否可查证？
   - 有没有遗漏关键信息？
3. **下半部分 — 真实纠错案例**（1-2个具体案例）
   - 格式："炭治郎给出了一个方案 → 义勇发现一个逻辑错误 → 重新修正后交付"
   - 展示出错了也不会直接发出去，而是内部修正

**叙事节奏：** 机制解释（建立理性信任）→ 真实纠错案例（建立感性信任）→ 对比单个AI的风险（强化差异化）

---

### 章节 D：怎么开始

**ID:** `start`  
**位置：** 全站最后一章（第11屏）  
**对应维度：** 怎么开始

**核心信息（用户看完应该得到的认知）：**
> "我知道下一步该做什么了，行动门槛很低。"

**内容结构：**
1. **主标题：** "准备好让鬼杀队出动了吗？"
2. **三步骤：**
   - Step 1: 发现你的任务（描述你面临的问题）
   - Step 2: 告诉天音（联系方式/入口）
   - Step 3: 坐等成果交付
3. **联系方式/入口区块**（根据实际情况填入）
4. **底部小字：** 首次任务说明（范围、响应时间等期望管理）

**设计要点：**
- 视觉上要"轻松"，不是商业感的CTA，而是"老朋友告诉你怎么找到他"的感觉
- 一定要有可点击的行动按钮，不能只是文字说明

## Part 3：每个新章节的小白用户体验审查标准

> **使用方法：** 炭治郎每写完一个章节，对照以下 checklist 自测。全部打勾才算完成，否则需要修改后重测。

---

### 章节 A 审查 Checklist：你有这些困扰吗？

**视觉体验**
- [ ] 痛点文字够大，第一眼就能看到关键词，不需要细读就有感觉
- [ ] 三种用户画像有视觉区分（颜色/图标），不会混成一团

**认知体验**
- [ ] 每个痛点描述不超过 15 字，读完一遍就能判断"说的是不是我"
- [ ] 痛点和用户画像之间有明确的过渡句，不会突然跳转让人迷惑
- [ ] 用户画像的描述用第一人称或第二人称（"你是...""像你这样的人..."），而不是第三人称（"这类用户..."）

**用户思考路径**
- [ ] 用户进入这章时会想："这说的是我吗？" — 章节在前3秒内给出答案（是/否）
- [ ] 用户离开这章时应该想："好像是我，我想看看它能做什么" — 不能让用户停留在"看完了不知道接下来干什么"

---

### 章节 B 审查 Checklist：鬼杀队能做什么？

**视觉体验**
- [ ] Tab 切换标签有明确的"当前选中"状态，不会让人搞不清看的是哪个场景
- [ ] 每个场景下的任务列表不超过 6 条，版面不拥挤

**认知体验**
- [ ] 每条任务描述用动词开头（"帮你写 / 帮你找 / 帮你做"），用户立刻明白"这是动作，不是概念"
- [ ] 全程零技术词汇：不出现"prompt""token""推理""向量"等词，替换为用户语言（"问问题""思考""记忆"）
- [ ] 每个场景标签至少有一个观众100%熟悉的任务（如"帮你写邮件"），让用户有"啊这个我用得上"的感觉

**用户思考路径**
- [ ] 用户进入这章时会想："具体能帮我做什么？" — 标签切换让用户主动探索，而不是被动阅读长列表
- [ ] 用户离开这章时应该想："好像有几个我真的需要" — 不能让用户觉得"这些都是大企业才用得上的"

---

### 章节 C 审查 Checklist：为什么可以信任我们？

**视觉体验**
- [ ] 质量门控流程图清晰显示4个步骤，每步之间有箭头方向，用户一眼看出是"流水线"不是"并列关系"
- [ ] 纠错案例用对话气泡或"修改前/修改后"对比格式，不用纯文字描述

**认知体验**
- [ ] 不用"审核""质量门控""合规"等偏商务/技术的词汇，改用"有人检查""先过一遍眼"等白话
- [ ] 义勇审核标准的折叠展开部分，折叠状态下只显示"义勇会检查什么？"一句话，好奇心驱动点开，不强迫阅读
- [ ] 纠错案例中明确出现"这个错误我们内部发现了，没有发给主公"的意思，让用户感知"有兜底"

**用户思考路径**
- [ ] 用户进入这章时会想："AI 会不会乱说话，我能信它吗？" — 章节直接回答这个顾虑，不拐弯抹角
- [ ] 用户离开这章时应该想："哦，它有自己的质检流程，不是裸跑的" — 不是"好厉害"而是"好靠谱"

---

### 章节 D 审查 Checklist：怎么开始

**视觉体验**
- [ ] 页面有且只有一个主 CTA 按钮（不能同时出现"联系我""试用""了解更多"三个按钮，让用户选择困难）
- [ ] 整体色调比前面章节"亮"一点，给人一种"轻松的开始"而不是"正式的申请"

**认知体验**
- [ ] 三个步骤每步不超过 8 字，用户30秒内读完整章
- [ ] 联系方式/入口处有明确的响应时间预期（"通常 X 小时内回复"），消除"发了没人理"的顾虑
- [ ] 没有任何需要填写复杂表格或注册账号的流程，门槛极低

**用户思考路径**
- [ ] 用户进入这章时会想："好想试试，但怎么联系？" — 章节在2秒内给出答案
- [ ] 用户离开这章时应该想："好，我知道怎么做了" — 不是"等我再想想"，而是"可以发消息了"

## 附：现有 brain 章节修改建议（记忆系统融入）

在现有「AI 大脑」章节的 Claude Sonnet/Opus 介绍下方，新增一个小模块：

```
📚 天音从不忘事
鬼杀队配备记忆系统——主公说过的每一件事、每个决策、每个项目背景，
天音都记得住。下次交代任务，不用重头解释。
```

无需技术细节，只传达"有记忆，不会每次重置"这个**用户价值**。

## 实施优先级建议

| 优先级 | 章节 | 理由 |
|--------|------|------|
| P0 🔴 | 章节 D（怎么开始）| 目前完全没有出口，是最大的转化漏洞 |
| P0 🔴 | 章节 A（痛点/目标用户）| 观众进入后第一个看到，决定要不要继续看 |
| P1 🟡 | 章节 B（能做什么）| 核心价值传达，但需要实际内容填充 |
| P2 🟢 | 章节 C（信任/质量）| 需要真实案例，可以先用"典型案例"占位 |
| P2 🟢 | brain 章节小修 | 工作量小，顺手完成 |
